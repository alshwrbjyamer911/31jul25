Introduction

This tutorial provides a detailed guide on using the Analog-to-Digital Converter (ADC) on the STM32F4 series microcontrollers with the STM32 HAL library. The STM32F4 series, based on the ARM Cortex-M4, features powerful ADCs capable of converting analog signals to digital with high precision. This tutorial covers ADC basics, HAL library functions with their prototypes, and a complete example to help embedded software engineers configure and use the ADC effectively.

ADC Basics on STM32F4

The STM32F4 series includes multiple ADC modules (typically ADC1, ADC2, ADC3) with the following key features:

  • Resolution: 12-bit (configurable to 10-bit, 8-bit, or 6-bit)
  • Channels: Up to 16 external channels, plus internal temperature sensor and Vref
  • Modes: Single, continuous, scan, and discontinuous modes
  • Conversion Trigger: Software, timer, or external event
  • DMA Support: For efficient data transfer
  • Sampling Time: Configurable to balance speed and accuracy

The ADC converts an analog voltage (0 to Vref, typically 3.3V) into a digital value using the formula:

Digital Value = (Analog Voltage / Vref) * (2Resolution - 1)

For a 12-bit ADC with Vref = 3.3V, a 1.65V input yields approximately 2048 (half of 4095).

HAL ADC APIs

The STM32 HAL library provides a comprehensive set of functions to configure and control the ADC. Below are the most commonly used HAL ADC APIs with their full function prototypes, arguments, and return types.

Initialization and Configuration

  • HAL_ADC_Init: Initializes the ADC with the specified configuration. 
    HAL_StatusTypeDef HAL_ADC_Init(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure containing configuration information.
    • Return: HAL_StatusTypeDef - Status of the operation (HAL_OK, HAL_ERROR, HAL_BUSY, HAL_TIMEOUT).
  • HAL_ADC_ConfigChannel: Configures a specific ADC channel, including rank and sampling time. 
    HAL_StatusTypeDef HAL_ADC_ConfigChannel(ADC_HandleTypeDef* hadc, ADC_ChannelConfTypeDef* sConfig)
    • Arguments:
      • hadc - Pointer to ADC handle structure.
      • sConfig - Pointer to channel configuration structure (e.g., channel number, rank, sampling time).
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADCEx_MultiModeConfigChannel: Configures multi-ADC mode (e.g., dual or triple mode). 
    HAL_StatusTypeDef HAL_ADCEx_MultiModeConfigChannel(ADC_HandleTypeDef* hadc, ADC_MultiModeTypeDef* multimode)
    • Arguments:
      • hadc - Pointer to ADC handle structure.
      • multimode - Pointer to multi-mode configuration structure (e.g., mode, DMA access).
    • Return: HAL_StatusTypeDef - Status of the operation.

Starting and Stopping Conversion

  • HAL_ADC_Start: Starts ADC conversion in polling mode. 
    HAL_StatusTypeDef HAL_ADC_Start(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADC_Start_IT: Starts ADC conversion in interrupt mode. 
    HAL_StatusTypeDef HAL_ADC_Start_IT(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADC_Start_DMA: Starts ADC conversion with DMA transfer. 
    HAL_StatusTypeDef HAL_ADC_Start_DMA(ADC_HandleTypeDef* hadc, uint32_t* pData, uint32_t Length)
    • Arguments:
      • hadc - Pointer to ADC handle structure.
      • pData - Pointer to the destination buffer for ADC data.
      • Length - Number of data items to transfer.
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADC_Stop: Stops ADC conversion in polling mode. 
    HAL_StatusTypeDef HAL_ADC_Stop(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADC_Stop_IT: Stops ADC conversion in interrupt mode. 
    HAL_StatusTypeDef HAL_ADC_Stop_IT(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: HAL_StatusTypeDef - Status of the operation.
  • HAL_ADC_Stop_DMA: Stops ADC conversion with DMA. 
    HAL_StatusTypeDef HAL_ADC_Stop_DMA(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: HAL_StatusTypeDef - Status of the operation.

Data Retrieval

  • HAL_ADC_GetValue: Retrieves the converted ADC value in polling mode. 
    uint32_t HAL_ADC_GetValue(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: uint32_t - The converted ADC value.
  • HAL_ADC_PollForConversion: Polls for conversion completion within a timeout period. 
    HAL_StatusTypeDef HAL_ADC_PollForConversion(ADC_HandleTypeDef* hadc, uint32_t Timeout)
    • Arguments:
      • hadc - Pointer to ADC handle structure.
      • Timeout - Timeout duration in milliseconds.
    • Return: HAL_StatusTypeDef - Status of the operation.

Callback Functions

  • HAL_ADC_ConvCpltCallback: Called when conversion is complete (interrupt mode). 
    void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: None (void).
  • HAL_ADC_ConvHalfCpltCallback: Called when half of the DMA buffer is filled. 
    void HAL_ADC_ConvHalfCpltCallback(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: None (void).
  • HAL_ADC_ErrorCallback: Called when an ADC error occurs. 
    void HAL_ADC_ErrorCallback(ADC_HandleTypeDef* hadc)
    • Arguments: hadc - Pointer to ADC handle structure.
    • Return: None (void).

Calibration

  • HAL_ADCEx_Calibration_Start: Starts ADC calibration to improve accuracy. 
    HAL_StatusTypeDef HAL_ADCEx_Calibration_Start(ADC_HandleTypeDef* hadc, uint32_t SingleDiff)
    • Arguments:
      • hadc - Pointer to ADC handle structure.
      • SingleDiff - Calibration mode (e.g., ADC_SINGLE_ENDED or ADC_DIFFERENTIAL_ENDED).
    • Return: HAL_StatusTypeDef - Status of the operation.

Each function returns a HAL_StatusTypeDef to indicate success (HAL_OK) or failure (HAL_ERROR, HAL_BUSY, HAL_TIMEOUT), except for callback functions and HAL_ADC_GetValue, which have specific return types.

Example Code

Below is a complete example of configuring ADC1 to read an analog input from channel 0 (PA0) in polling mode. The code assumes you are using STM32CubeMX-generated code as a base.

#include "main.h"

ADC_HandleTypeDef hadc1;

void SystemClock_Config(void);
static void MX_GPIO_Init(void);
static void MX_ADC1_Init(void);

int main(void) {
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_ADC1_Init();

    uint32_t adc_value = 0;

    while (1) {
        HAL_ADC_Start(&hadc1);
        HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY);
        adc_value = HAL_ADC_GetValue(&hadc1);
        HAL_ADC_Stop(&hadc1);

        // Convert ADC value to voltage (Vref = 3.3V, 12-bit resolution)
        float voltage = (adc_value * 3.3f) / 4095.0f;

        // Add your processing here (e.g., send to UART)
        HAL_Delay(500); // Delay for 500ms
    }
}

Configuration Steps

Follow these steps to configure the ADC using STM32CubeMX and HAL:

  1. Enable ADC: In STM32CubeMX, select ADC1 (or desired ADC) and enable it in analog mode for the desired pin (e.g., PA0 for channel 0).
  2. Configure ADC Parameters:
    • Clock Prescaler: Set to divide the ADC clock (e.g., ADC_CLOCK_SYNC_PCLK_DIV4).
    • Resolution: Choose 12-bit for maximum precision.
    • Conversion Mode: Select single or continuous mode.
    • Trigger: Set to software start for manual control.
    • Sampling Time: Choose a value (e.g., 480 cycles) to ensure accurate sampling.
  3. Generate Code: Use STM32CubeMX to generate the initialization code.
  4. Calibrate ADC: Call HAL_ADCEx_Calibration_Start before starting conversions.
  5. Start Conversion: Use HAL_ADC_Start (polling), HAL_ADC_Start_IT (interrupt), or HAL_ADC_Start_DMA (DMA).
  6. Read Data: Use HAL_ADC_GetValue (polling) or process data in callbacks (interrupt/DMA).

Tips and Best Practices

  • Calibration: Always calibrate the ADC to minimize offset errors.
  • Sampling Time: Increase sampling time for high-impedance sources to ensure accurate conversions.
  • DMA: Use DMA for continuous or multi-channel conversions to reduce CPU load.
  • Interrupt Mode: Prefer interrupts over polling for real-time applications.
  • Vref: Ensure a stable reference voltage (e.g., 3.3V) for accurate conversions.
  • Error Handling: Check return values of HAL functions and implement error handling.
